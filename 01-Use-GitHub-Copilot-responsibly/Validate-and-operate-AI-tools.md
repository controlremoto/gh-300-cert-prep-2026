# Validate and Operate AI Tools

## Learning Points

### Explain the Need to Validate AI Output

| Reason                | Description                                                                 |
| :---                  | :---                                                                        |
| Error Prevention      | AI may generate code with bugs, vulnerabilities, or logical errors.         |
| Compliance            | Ensures outputs meet legal, regulatory, and organizational standards.       |
| Appropriateness       | Validates that content is suitable for the intended context.                |
| Security              | Prevents introduction of insecure or unsafe code.                           |

**Exam Tricky Points:**
- Human oversight is always required—AI suggestions are not guaranteed to be correct.
- Automated validation tools can help, but do not replace manual review.

---

### Identify How to Operate GitHub Copilot Responsibly

| Practice                        | Description                                                      |
| :---                            | :---                                                             |
| Review Suggestions               | Always review and test Copilot’s code before using it.           |
| Respect Licensing                | Attribute and respect licenses for generated code.               |
| Avoid Sensitive Data             | Do not use Copilot for confidential or regulated information.    |
| Provide Feedback                 | Report inappropriate or incorrect suggestions to GitHub.         |

**Exam Tricky Points:**
- Copilot suggestions may include code similar to public repositories—always check for licensing.
- Responsible operation includes disabling Copilot in sensitive projects if required.
